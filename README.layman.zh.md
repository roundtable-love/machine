# 清晰沟通协议: 1.0

**状态:** 已完成
**基础语言:** 中文 (Chinese)
**规则:** 严格、清晰的定义

---

## 1. 简介
本协议是清晰意图和肢体表达的完美结合，确保信息零丢失。

## 2. 基础连接：设定氛围
* **氛围测试 (Vibe-Ping):** 一个简单的问候（如：“嗨”）以测试对方的注意力。
* **共鸣:** 双方思想同步的状态。
* **缓冲:** 消除压力和自负，创造平静的对话环境。

## 3. 非语言信号：肢体语言与打断
* **彻底打断:** 一个重大的肢体动作（如竖中指），立即停止所有沟通。
* **一致性:** 肢体语言必须与言辞一致。
* **全局终止信号:** 严重的打断意味着当前话题结束。

## 4. 思想翻译：适应听众
* **核心意图:** 真实意图（必须、禁止、可以）。
* **调整信息:** 根据听众（专家 vs. 新手）改变表达方式。
* **心理负担:** 如果听众不堪重负，必须暂停对话。

## 5. 案例研究: NVIDIA_HALT_2012
Linus Torvalds 由于缺乏合作，对 NVIDIA 使用了“彻底打断”的动作，终止了合作关系。

## 6. 架构
想法 -> 清晰规则 -> 听众适应。

## 7. 严格规则
* **非黑即白:** 只有明确的“是”或“否”。
* **不用“应该”:** 只能用“必须”或“可以”。
* **零偏差:** 核心意义必须保持不变。

## 8. 元数据
* **语言代码:** zh
* **协议:** MCH-LOGIC-1.0
