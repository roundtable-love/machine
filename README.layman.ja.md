# 明確なコミュニケーションプロトコル: 1.0

**ステータス:** 完了
**基本言語:** 日本語 (Japanese)
**ルール:** 厳格で明確な定義

---

## 1. はじめに
本プロトコルは明確な意図と身体的表現の完璧な組み合わせであり、情報の損失を防ぎます。

## 2. 基本接続: 雰囲気を整える
* **Vibe-Ping:** 注意を引くための簡単な挨拶（例: 「やあ」）。
* **共鳴:** 双方が同じ認識を持っている状態。
* **減衰:** ストレスやエゴを取り除き、落ち着いた環境を作る。

## 3. 非言語シグナル: ボディランゲージ
* **完全な中断:** 会話を即座に停止させる大きな身体的ジェスチャー。
* **一貫性:** ボディランゲージと言葉は一致していなければならない。
* **停止シグナル:** 重大な中断はセッションの終了を意味する。

## 4. 思考の翻訳: 聞き手への適応
* **コアの意味:** 本当の意図（MUST、MUST NOT、MAY）。
* **メッセージの適応:** 聞き手（専門家 vs 初心者）に合わせて表現を変える。
* **精神的負荷:** 聞き手がキャパオーバーなら会話を一時停止する。

## 5. ケーススタディ: NVIDIA_HALT_2012
Linus Torvaldsは協力の欠如を理由に、NVIDIAに対して「完全な中断」のジェスチャーを使用し、関係を断ち切った。

## 6. アーキテクチャ
思考 -> 明確なルール -> 聞き手への適応。

## 7. 厳格なルール
* **白黒はっきり:** 「はい」か「いいえ」のみ。
* **「Should(すべき)」の禁止:** MUSTかMAYのみを使用する。
* **情報の漏れゼロ:** 意味は完全に同じでなければならない。
