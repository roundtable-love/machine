# 명확한 의사소통 프로토콜: 1.0

**상태:** 완료
**기본 언어:** 한국어 (Korean)
**규칙:** 엄격하고 명확한 정의

---

## 1. 소개
이 프로토콜은 명확한 의도와 신체적 표현의 완벽한 결합으로, 정보 손실을 방지합니다.

## 2. 기본 연결: 분위기 조성
* **분위기 확인 (Vibe-Ping):** 주의를 끌기 위한 간단한 인사 (예: "안녕").
* **공명:** 양측이 같은 생각을 공유하는 상태.
* **진정:** 스트레스와 자존심을 줄여 차분한 환경 조성.

## 3. 비언어적 신호: 바디 랭귀지
* **완전한 중단:** 대화를 즉시 멈추게 하는 강력한 신체적 제스처.
* **일관성:** 바디 랭귀지는 말과 일치해야 합니다.
* **중지 신호:** 심각한 중단은 대화 세션의 종료를 의미합니다.

## 4. 생각의 번역: 청중에게 맞추기
* **핵심 의미:** 진정한 의도 (해야 한다, 해서는 안 된다, 할 수 있다).
* **메시지 조정:** 듣는 사람(전문가 vs 초보자)에 따라 표현 방식 변경.
* **정신적 부담:** 듣는 이가 버거워하면 대화를 일시 중지.

## 5. 사례 연구: NVIDIA_HALT_2012
리누스 토발즈(Linus Torvalds)는 협력 부족을 이유로 NVIDIA에 "완전한 중단" 제스처를 사용하여 관계를 끊었습니다.

## 6. 아키텍처
생각 -> 명확한 규칙 -> 청중 맞춤 변환.

## 7. 엄격한 규칙
* **흑백 논리:** 오직 "예" 또는 "아니오".
* **"해야 할 것 같다" 금지:** "반드시 해야 한다" 또는 "할 수 있다"만 사용.
* **정보 누락 제로:** 의미는 정확히 동일하게 유지되어야 함.
